# ติดตั้งเซิร์ฟเวอร์ซิงก์ + AI

ทำครั้งเดียวราว 15 นาที ใช้บัญชี Cloudflare ฟรี ไม่ต้องผูกบัตร

เซิร์ฟเวอร์ตัวนี้ทำสองอย่างพร้อมกัน — ซิงก์ข้อมูลระหว่างมือถือทุกเครื่อง และถือ API key ของ Anthropic ไว้ให้ AI อ่านฉลากทำงานได้

---

## 1. สร้างฐานข้อมูล D1

1. เข้า https://dash.cloudflare.com → **Storage & Databases → D1 SQL Database** → Create
2. ตั้งชื่อ `suru` → Create

ไม่ต้องสร้างตารางเอง Worker สร้างให้อัตโนมัติตอนถูกเรียกครั้งแรก

## 2. สร้าง Worker

1. **Workers & Pages → Create → Workers → Create Worker**
2. ตั้งชื่อ เช่น `suru` → Deploy
3. กด **Edit code** ลบโค้ดตัวอย่างทิ้ง วางเนื้อหาจาก `worker.js` ทั้งหมด → Deploy

## 3. ผูกฐานข้อมูลเข้ากับ Worker

ที่หน้า Worker → **Settings → Bindings → Add binding → D1 database**

| ช่อง | ค่า |
|---|---|
| Variable name | `DB` |
| D1 database | `suru` |

ชื่อตัวแปรต้องเป็น `DB` ตัวใหญ่ทั้งสองตัว ไม่งั้นโค้ดหาไม่เจอ

## 4. ตั้งค่าตัวแปร

หน้าเดียวกัน → **Variables and Secrets**

| ชื่อ | ประเภท | ค่า |
|---|---|---|
| `APP_TOKEN` | Secret | รหัสผ่านที่ตั้งเอง ทุกเครื่องต้องกรอกให้ตรงกัน |
| `ALLOWED_ORIGIN` | Variable | `https://nudanica.github.io` |
| `ANTHROPIC_API_KEY` | Secret | คีย์จาก https://console.anthropic.com — ไม่ใส่ก็ได้ถ้ายังไม่ใช้ AI |

กด **Deploy** อีกครั้งหลังตั้งค่าเสร็จ

## 5. ตั้งค่าในแอปทุกเครื่อง

เปิดแอป → ปุ่ม ⚙ มุมขวาบน → กรอก

- **ที่อยู่เซิร์ฟเวอร์** — URL ของ Worker เช่น `https://suru.ชื่อบัญชี.workers.dev`
- **รหัสผ่านเซิร์ฟเวอร์** — ค่าเดียวกับ `APP_TOKEN`
- **ชื่อเครื่องนี้** — เช่น "มือถือพี่หนู" จะได้รู้ว่าใครถ่ายรายการไหน

## 6. เข้าร่วมทริปเดียวกัน

เครื่องแรกสร้างทริป แล้วเปิด ⚙ จะเห็น **รหัสทริป** เป็นตัวอักษร 6 ตัว เช่น `TKY7F3`

เครื่องที่เหลือกดที่ชื่อทริปบนหัวจอ → ช่อง "เข้าร่วมทริปที่มีอยู่" → กรอกรหัส → เข้าร่วม

จากนั้นทุกเครื่องจะเห็นของกันภายในไม่กี่วินาที

---

## ทดสอบว่าใช้ได้

```bash
curl -X POST https://suru.ชื่อบัญชี.workers.dev/sync \
  -H "Content-Type: application/json" \
  -H "X-App-Token: รหัสผ่านของคุณ" \
  -d '{"trip":"TEST","since":0,"changes":[]}'
```

ได้ `{"now":...,"records":[]}` กลับมาคือใช้ได้

## การซิงก์ทำงานอย่างไร

ทุกเครื่องบันทึกลงเครื่องตัวเองก่อนเสมอ แล้วค่อยส่งขึ้นเซิร์ฟเวอร์เป็นเบื้องหลัง ถ่ายในชั้นใต้ดินที่ไม่มีสัญญาณก็บันทึกได้ตามปกติ พอกลับมามีเน็ตระบบส่งเองอัตโนมัติ จุดสีบนปุ่ม ⚙ บอกสถานะ

- เขียว — ส่งขึ้นเซิร์ฟเวอร์ครบแล้ว
- เหลือง — มีรายการรอส่ง
- เทา — ยังไม่ได้ตั้งค่าเซิร์ฟเวอร์ ใช้แบบเครื่องเดียว

ดึงข้อมูลใหม่ทุก 15 วินาทีระหว่างเปิดแอปอยู่ และดึงทันทีเมื่อสลับกลับมาที่แอป

**การแก้ไขและลบ** ส่งต่อไปทุกเครื่องเหมือนกัน ถ้าสองเครื่องแก้รายการเดียวกันพร้อมกัน ระบบยึดฉบับที่แก้ทีหลัง วัดจากนาฬิกาของเครื่องที่แก้ ควรตั้งเวลาอัตโนมัติไว้ทุกเครื่อง

**การลบ** ไม่ได้ลบแถวออกจริง แต่ทำเครื่องหมายไว้ เพื่อให้เครื่องอื่นรู้ว่าต้องลบตาม ถ้าอยากล้างของเก่าทิ้งจริง ๆ ให้ลบทั้งทริปออกจาก D1 ผ่านหน้า Console ของฐานข้อมูล

## โควตาที่ใช้

ฟรีทั้งหมดสำหรับการใช้งานระดับนี้ ตัวเลขที่ Cloudflare ให้ในแผนฟรี

| ทรัพยากร | โควตาฟรี | ที่เราน่าจะใช้ |
|---|---|---|
| Workers | 100,000 คำขอต่อวัน | 5 เครื่องเปิดแอปทั้งวันราว 15,000 |
| D1 เขียน | 100,000 แถวต่อวัน | หลักร้อยถึงหลักพัน |
| D1 พื้นที่ | 5 GB | รูปใบละราว 120 KB เก็บได้ราว 40,000 ใบ |

ถ้าเลยโควตา D1 จะหยุดรับคำขอจนถึงเที่ยงคืน UTC ข้อมูลในเครื่องยังอยู่ครบ ซิงก์ต่อได้วันถัดไป

ตรวจตัวเลขล่าสุดได้ที่ https://developers.cloudflare.com/d1/platform/pricing/

## ความปลอดภัย

`APP_TOKEN` กับ `ALLOWED_ORIGIN` คือสิ่งเดียวที่กันคนนอก ใครรู้ URL กับรหัสผ่านก็เข้าถึงข้อมูลทริปได้ ถ้ารหัสหลุดให้เปลี่ยนค่าใน Cloudflare แล้วแก้ในแอปทุกเครื่อง

รหัสทริปสั้นเพียง 6 ตัวอักษร เพราะต้องพิมพ์ง่ายหน้าร้าน ความปลอดภัยจริงมาจาก `APP_TOKEN` ไม่ใช่จากรหัสทริป
