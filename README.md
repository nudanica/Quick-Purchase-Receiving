# SURU Quick Purchase List

เว็บแอปมือถือสำหรับบันทึกรายการซื้อสินค้าระหว่างเดินทางไปต่างประเทศ
ออกแบบให้บันทึกได้เร็วที่สุดตอนยืนเลือกของอยู่หน้าชั้นวาง หลายเครื่องช่วยกันถ่ายในทริปเดียวกันได้ แล้วส่งออกเป็น PDF ไว้ปริ้นตรวจรับของเมื่อสินค้ามาถึงไทย

**เปิดใช้งาน:** https://nudanica.github.io/Quick-Purchase-Receiving/

---

## แนวคิด

ตอนซื้อสินค้าจริงไม่มีเวลาพิมพ์รายละเอียด ผู้ใช้จึงกรอกแค่ 3 อย่าง

1. ถ่ายรูปสินค้า
2. ต้นทุนต่อชิ้น
3. จำนวนที่ซื้อ

ที่เหลือระบบเติมให้เอง — วันที่ เวลา ทริป สกุลเงิน ยอดรวม ชื่อเครื่องที่บันทึก และชื่อสินค้าที่ AI อ่านจากแพ็กเกจ
กดบันทึกแล้วกล้องเปิดต่อทันทีสำหรับสินค้าชิ้นถัดไป

การรับสินค้าไม่ทำในระบบ — ปริ้น PDF ออกมาแล้วใช้ปากกาติ๊กและเขียนจำนวนบนกระดาษ

## PDF ที่ระบบสร้าง

**1. ใบตรวจรับสินค้า** — รวมสินค้าชนิดเดียวกันเป็นหนึ่งรายการ 6 รายการต่อหน้า A4

ลำดับ · รูปสินค้า · ชื่อสินค้า · จำนวนที่สั่ง (ตัวเลขใหญ่) · ช่องติ๊กว่าครบ · ช่องเขียนจำนวนที่ได้รับจริง · ช่องเขียนจำนวนขาดหรือเกิน · ช่องหมายเหตุ พร้อมช่องเซ็นชื่อผู้ตรวจรับท้ายเอกสาร

**2. ใบรายละเอียดการซื้อ** — แยกทุกครั้งที่กดบันทึก 8 แถวต่อหน้า A4

ลำดับ · รูป · ชื่อสินค้าและวันเวลาและเครื่องที่บันทึก · ต้นทุนต่อชิ้น · จำนวน · ยอดรวม · ค่าประมาณเป็นเงินบาท · ช่องหมายเหตุ

รายการที่ยังไม่มีชื่อจะเว้นเส้นประไว้ให้เขียนด้วยปากกาบนกระดาษ
ก่อนสร้างไฟล์ ระบบจะดึงรูปจากเครื่องอื่นในทริปมาให้ครบก่อน

## การรวมสินค้าซ้ำ

ให้คะแนนความน่าจะเป็นสินค้าชนิดเดียวกันจากสามสัญญาณ

- บาร์โค้ดตรงกัน — 0.99
- ชื่อสินค้าใกล้เคียง — น้ำหนัก 0.65 (เทียบคำซ้อนกันหลัง normalize)
- รูปคล้ายกัน — น้ำหนัก 0.35 (average hash 8×8 เทียบ Hamming distance)

เกิน 0.62 จะขึ้นการ์ดแนะนำพร้อมรูปทั้งคู่ ผู้ใช้เลือก "รวมเป็นรายการเดียว" หรือ "คนละอย่าง"
เมื่อรวมแล้วระบบคิดจำนวนรวมและต้นทุนเฉลี่ยถ่วงน้ำหนักให้ พร้อมเก็บรายการซื้อแต่ละครั้งไว้ย้อนดูและแก้ไขได้

## ข้อมูลเก็บที่ไหน

**บนเครื่องเสมอ** — IndexedDB ของเบราว์เซอร์ อยู่ต่อเมื่อรีเฟรช ปิดแท็บ หรือปิดเครื่อง
แอปเรียก `navigator.storage.persist()` ตอนเปิดเพื่อลดโอกาสถูกลบอัตโนมัติ
บนเบราว์เซอร์ที่ห้ามเก็บข้อมูลเช่นโหมดส่วนตัว แอปจะเตือนว่าข้อมูลจะหายเมื่อปิดแท็บ

**บนเซิร์ฟเวอร์ ถ้าตั้งค่าไว้** — Cloudflare D1 ของบัญชีคุณเอง ใช้เพื่อให้ทุกเครื่องเห็นตรงกันและกู้คืนได้เมื่อมือถือหาย

โครงสร้างข้อมูล

```
trip   { id, name, currency, rate, updated, deleted }        // id คือรหัสทริป
group  { id, tripId, name, skip[], updated, deleted }        // สินค้าหนึ่งชนิด
entry  { id, tripId, groupId, cost, qty, total, createdAt,
         thumb, hash, by, ai{...}, updated, deleted }        // การซื้อหนึ่งครั้ง
```

รูปย่อเดินทางไปกับรายการเพื่อให้หน้าจอขึ้นทันที ส่วนรูปเต็มดึงมาเฉพาะตอนเปิดดูหรือสร้าง PDF

## AI อ่านฉลาก

ในหน้าต่าง Claude ใช้งานได้ทันที บนลิงก์เว็บต้องตั้งเซิร์ฟเวอร์ก่อน เพราะเบราว์เซอร์เรียก Anthropic API ตรง ๆ ไม่ได้ และการฝัง API key ไว้ในหน้าเว็บเท่ากับแจกคีย์ให้ทุกคน

ถ้าไม่ตั้งเซิร์ฟเวอร์ แอปยังใช้ได้ครบทุกอย่างในโหมดเครื่องเดียว เพียงแต่ต้องพิมพ์ชื่อสินค้าเอง

## เทคโนโลยี

ไฟล์ HTML เดียว ไม่มี build step ไม่มี dependency ที่ต้องติดตั้ง

- Vanilla JavaScript
- IndexedDB สำหรับข้อมูลบนเครื่อง
- Cloudflare Workers + D1 สำหรับซิงก์ ใช้หลัก last-write-wins ต่อรายการ พร้อม tombstone สำหรับการลบ
- Anthropic Messages API สำหรับอ่านฉลากสินค้า
- `BarcodeDetector API` อ่านบาร์โค้ดจากรูป เมื่อเบราว์เซอร์รองรับ
- html2canvas + jsPDF สร้าง PDF ฝั่งเครื่องผู้ใช้ (rasterize ทำให้ฟอนต์ไทยและญี่ปุ่นไม่เพี้ยน) มีทางสำรองเป็นหน้าปริ้นของเบราว์เซอร์

## ข้อจำกัดที่รู้อยู่

- ถ้าสองเครื่องแก้รายการเดียวกันพร้อมกัน ระบบยึดฉบับที่แก้ทีหลัง โดยวัดจากนาฬิกาของเครื่องนั้น ควรตั้งเวลาอัตโนมัติไว้ทุกเครื่อง
- เรทเงินบาทกรอกเองตอนสร้างทริป ไม่ได้ดึงเรทสด
- ใครมี URL เซิร์ฟเวอร์กับ `APP_TOKEN` ก็เข้าถึงข้อมูลทุกทริปได้ ยังไม่มีระบบผู้ใช้แยกรายคน
- ข้อความใน PDF เลือกคัดลอกไม่ได้ เพราะแต่ละหน้าถูกแปลงเป็นภาพก่อนประกอบเป็นไฟล์
