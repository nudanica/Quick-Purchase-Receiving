# SURU Quick Purchase List

เว็บแอปมือถือสำหรับบันทึกรายการซื้อสินค้าระหว่างเดินทางไปต่างประเทศ
ออกแบบให้บันทึกได้เร็วที่สุดตอนยืนเลือกของอยู่หน้าชั้นวาง แล้วส่งออกเป็น PDF ไว้ปริ้นตรวจรับของเมื่อสินค้ามาถึงไทย

**เปิดใช้งาน:** https://nudanica.github.io/Quick-Purchase-Receiving/

---

## แนวคิด

ตอนซื้อสินค้าจริงไม่มีเวลาพิมพ์รายละเอียด ผู้ใช้จึงกรอกแค่ 3 อย่าง

1. ถ่ายรูปสินค้า
2. ต้นทุนต่อชิ้น
3. จำนวนที่ซื้อ

ที่เหลือระบบเติมให้เอง — วันที่ เวลา ทริป สกุลเงิน ยอดรวม และชื่อสินค้าที่ AI อ่านจากแพ็กเกจ
กดบันทึกแล้วกล้องเปิดต่อทันทีสำหรับสินค้าชิ้นถัดไป

การรับสินค้าไม่ทำในระบบ — ปริ้น PDF ออกมาแล้วใช้ปากกาติ๊กและเขียนจำนวนบนกระดาษ

## ฟังก์ชัน

| กลุ่ม | รายละเอียด |
|---|---|
| ทริป | สร้างและสลับทริป แยกสกุลเงินและเรทเงินบาทต่อทริป |
| บันทึกเร็ว | ถ่ายรูป ใส่ต้นทุน ใส่จำนวน คำนวณยอดรวมทันที บังคับแค่ 3 ช่องนี้ |
| AI อ่านฉลาก | อ่านข้อความบนแพ็กเกจทั้งภาษาญี่ปุ่นและอังกฤษ เสนอแบรนด์ ชื่อสินค้า สูตร กลิ่น รส ขนาดบรรจุ และเลขบาร์โค้ด ทำงานเบื้องหลังหลังบันทึก ไม่หน่วงขั้นตอนบันทึก |
| รวมสินค้าซ้ำ | แนะนำคู่ที่น่าจะเป็นชนิดเดียวกันจากบาร์โค้ด ชื่อ และความคล้ายของรูป ผู้ใช้ยืนยันก่อนรวมเสมอ |
| สรุปทริป | รูป ชื่อ จำนวนรวม ต้นทุนเฉลี่ยต่อชิ้น ยอดต้นทุนรวม และจำนวนครั้งที่ซื้อ |
| ส่งออก PDF | ใบตรวจรับสินค้า และใบรายละเอียดการซื้อทุกครั้ง |

## PDF ที่ระบบสร้าง

**1. ใบตรวจรับสินค้า** — รวมสินค้าชนิดเดียวกันเป็นหนึ่งรายการ 6 รายการต่อหน้า A4

ลำดับ · รูปสินค้า · ชื่อสินค้า · จำนวนที่สั่ง (ตัวเลขใหญ่) · ช่องติ๊กว่าครบ · ช่องเขียนจำนวนที่ได้รับจริง · ช่องเขียนจำนวนขาดหรือเกิน · ช่องหมายเหตุ พร้อมช่องเซ็นชื่อผู้ตรวจรับท้ายเอกสาร

**2. ใบรายละเอียดการซื้อ** — แยกทุกครั้งที่กดบันทึก 8 แถวต่อหน้า A4

ลำดับ · รูป · ชื่อสินค้าและวันเวลา · ต้นทุนต่อชิ้น · จำนวน · ยอดรวม · ค่าประมาณเป็นเงินบาท · ช่องหมายเหตุ

รายการที่ยังไม่มีชื่อจะเว้นเส้นประไว้ให้เขียนด้วยปากกาบนกระดาษ

## การรวมสินค้าซ้ำ

ให้คะแนนความน่าจะเป็นสินค้าชนิดเดียวกันจากสามสัญญาณ

- บาร์โค้ดตรงกัน — 0.99
- ชื่อสินค้าใกล้เคียง — น้ำหนัก 0.65 (เทียบคำซ้อนกันหลัง normalize)
- รูปคล้ายกัน — น้ำหนัก 0.35 (average hash 8×8 เทียบ Hamming distance)

เกิน 0.62 จะขึ้นการ์ดแนะนำพร้อมรูปทั้งคู่ ผู้ใช้เลือก "รวมเป็นรายการเดียว" หรือ "คนละอย่าง"
เมื่อรวมแล้วระบบคิดจำนวนรวมและต้นทุนเฉลี่ยถ่วงน้ำหนักให้ พร้อมเก็บรายการซื้อแต่ละครั้งไว้ย้อนดูได้

## โครงสร้างข้อมูล

```
trip    { id, name, currency, rate, createdAt }
group   { id, tripId, name, skip[] }          // สินค้าหนึ่งชนิด
entry   { id, tripId, groupId, cost, qty, total, createdAt,
          currency, rate, thumb, hash, pi, ai{...} }   // การซื้อหนึ่งครั้ง
```

ข้อมูลเก็บบนเครื่องผู้ใช้ผ่าน persistent storage แยก metadata ออกจากรูปเต็ม
รูปเต็มถูกมัดเป็นก้อนละ 8 รูปเพื่อไม่ให้ชนเพดานขนาดต่อ key และมี thumbnail อยู่ใน metadata เพื่อให้หน้ารายการแสดงผลทันที

## เทคโนโลยี

ไฟล์ HTML เดียว ไม่มี build step ไม่มี dependency ที่ต้องติดตั้ง

- Vanilla JavaScript
- Anthropic Messages API สำหรับอ่านฉลากสินค้า
- `BarcodeDetector API` อ่านบาร์โค้ดจากรูป เมื่อเบราว์เซอร์รองรับ
- html2canvas + jsPDF สร้าง PDF ฝั่งเครื่องผู้ใช้ (rasterize ทำให้ฟอนต์ไทยและญี่ปุ่นไม่เพี้ยน) มีทางสำรองเป็นหน้าปริ้นของเบราว์เซอร์

---

## ติดตั้งเอง

โคลนแล้วเปิด `index.html` ผ่าน static server ใดก็ได้ กล้องต้องรันบน `https` หรือ `localhost` เท่านั้น

```bash
python3 -m http.server 8000
```

เปิด http://localhost:8000

## ข้อจำกัดที่รู้อยู่

- ข้อมูลเก็บบนเครื่องที่ใช้บันทึก ยังไม่มีการซิงก์ข้ามอุปกรณ์
- เรทเงินบาทกรอกเองตอนสร้างทริป ไม่ได้ดึงเรทสด
- ยังไม่มีการแก้ไขหรือลบรายการซื้อรายครั้งหลังบันทึกไปแล้ว
